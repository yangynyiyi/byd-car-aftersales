## 五、组员接手指南

### 5.1 环境准备

```
1. 安装 JDK 17+
2. 安装 MySQL 8.0，创建数据库 byd_aftersales
3. 安装 Node.js（前端 Vue 3 运行需要）
4. IDEA 打开 byd-aftersales 子目录（不是仓库根目录）
5. 右键 BydAftersalesApplication → Run
```

### 5.2 代码结构

```
byd-aftersales/src/main/java/com/byd/
├── aftersales/           ← B模块（用户/车辆/预约/故障）
│   ├── common/           ← ApiResponse、GlobalExceptionHandler
│   ├── controller/       ← REST 接口
│   ├── dao/              ← JDBC 数据访问
│   ├── domain/           ← 实体类
│   └── service/          ← 业务逻辑
├── car/
│   ├── agent/            ← D模块（Agent 诊断）
│   │   ├── client/       ← AiAgentClient（调通义千问）
│   │   ├── controller/   ← AgentDiagnosisController
│   │   ├── dao/          ← AgentDiagnosisDao、FaultRecordDao 等
│   │   ├── model/        ← AgentDiagnosis、AgentDiagnosisResult
│   │   └── service/      ← AgentDiagnosisService
│   ├── battery/          ← D模块（电池健康）
│   │   ├── controller/   ← BatteryHealthController
│   │   ├── dao/          ← BatteryHealthDao
│   │   ├── model/        ← BatteryHealthRecord
│   │   └── service/      ← BatteryHealthService
│   ├── config/           ← RedisConfig、CacheService
│   ├── part/             ← C模块（备件管理）
│   ├── settlement/       ← C模块（结算管理）
│   ├── transaction/      ← C模块（工单完工事务）
│   └── workorder/        ← C模块（工单管理）
```

### 5.3 开发新功能

1. **新增接口**：在对应模块的 controller 包下新建或修改 Controller
2. **新增表**：在 `resources/db/` 下写 DDL，手动在 MySQL 执行
3. **新增实体**：在对应的 model/domain/entity 包下建类
4. **新增 DAO**：继承 `JdbcDaoSupport`，参考现有 DAO 写法
5. **返回类型**：统一用 `ApiResponse<T>`，成功用 `ApiResponse.ok(data)`，失败用 `ApiResponse.fail(code, msg)`

### 5.4 注意事项

- `ApiResponse.fail()` 已改为泛型，可直接返回 `ApiResponse<BatteryHealthRecord>` 等类型
- `operation_log` 当前是 MySQL 表，不再依赖 MongoDB
- 模块 D 的 Agent 诊断需要能访问通义千问 API（外网），本地测试需确保网络通畅
- Bean 名称冲突：模块 D 的 `FaultRecordDao`、`VehicleDao`、`WorkOrderDao` 已加 `@Qualifier` 前缀，新增 DAO 时注意避免同名冲突
- `application.yml` 中 MongoDB 自动配置已排除，当前项目默认不启用 MongoDB
